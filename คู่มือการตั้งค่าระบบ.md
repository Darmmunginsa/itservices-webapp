# คู่มือการตั้งค่าระบบ IT Services Helpdesk

## 1. Email Templates (การแจ้งเตือนทางอีเมล)

### ตำแหน่ง
**Admin → Email Templates**

### รายการ EventKey ที่มี

| EventKey | ส่งเมื่อ | ส่งถึง |
|---|---|---|
| `ticket_created` | สร้าง Ticket ใหม่ | ลูกค้า + Agent ที่รับงาน |
| `task_assigned` | สร้าง Task ใหม่ | Agent ที่รับงาน |
| `ticket_status_changed` | เปลี่ยนสถานะ Ticket | ลูกค้า |
| `comment_added` | เพิ่ม Comment (External) | ลูกค้า |
| `leave_requested` | ส่งคำขอลา | ผู้อนุมัติ |
| `leave_decision` | อนุมัติ/ปฏิเสธการลา | พนักงานผู้ขอลา |

### วิธีแก้ไข Template
1. คลิกไอคอน ✏️ ข้าง Template ที่ต้องการ
2. แก้ไข **Subject** และ **Body** (รองรับ HTML)
3. คลิก **บันทึก**

### ตัวแปร (Variables) ที่ใช้ใน Template

ใช้รูปแบบ `{{ชื่อตัวแปร}}` ใน Subject หรือ Body

**Ticket / Task:**
| ตัวแปร | ความหมาย |
|---|---|
| `{{ticket_id}}` | หมายเลข Ticket |
| `{{ticket_title}}` | ชื่อ Ticket |
| `{{ticket_status}}` | สถานะปัจจุบัน |
| `{{priority}}` | ความสำคัญ |
| `{{category}}` | ประเภท |
| `{{customer_name}}` | ชื่อลูกค้า |
| `{{assigned_name}}` | ชื่อ Agent |
| `{{link}}` | ลิงก์เข้าสู่ระบบ |

**Leave:**
| ตัวแปร | ความหมาย |
|---|---|
| `{{requester_name}}` | ชื่อพนักงานผู้ขอลา |
| `{{leave_type}}` | ประเภทการลา |
| `{{leave_date}}` | วันที่ลา |
| `{{leave_status}}` | อนุมัติ / ไม่อนุมัติ |
| `{{approver_name}}` | ชื่อผู้อนุมัติ |
| `{{link}}` | ลิงก์เข้าสู่ระบบ |

### เปิด/ปิด Template
- คลิกปุ่ม Toggle (สีเขียว = เปิด / สีแดง = ปิด) ข้าง Template นั้น
- **แนะนำ: ปิดทุก Template ในช่วงทดสอบระบบ แล้วค่อยเปิดทีละตัว**

## 3. Agent Profiles

### ตำแหน่ง
**Admin → Agent Profiles** (SP List: `HD_AgentProfiles`)

### ฟิลด์สำคัญ
| ฟิลด์ | คำอธิบาย |
|---|---|
| Title | ชื่อ Agent (แสดงใน Dropdown) |
| EmailText | อีเมล (ใช้ส่ง notification + filter งาน) |
| Role | `Agent` / `Admin` / `Boss` |
| IsActive | เปิด/ปิดการใช้งาน |

> **หมายเหตุ:** EmailText ต้องตรงกับ Microsoft 365 account ของผู้ใช้

## 4. Leave Quota (โควต้าวันลา)

### ตำแหน่ง
**Admin → Leave Quota** (SP List: `HD_LeaveQuota`)

### วิธีตั้งค่า
- แต่ละแถว = พนักงาน 1 คน × ประเภทการลา 1 ประเภท
- ตัวอย่าง: สมชาย / ลาพักร้อน / 10 วัน

| ฟิลด์ | คำอธิบาย |
|---|---|
| Title | ประเภทการลา (ต้องตรงกับ `leave_type` ใน HD_Options) |
| EmployeeEmail | อีเมลพนักงาน |
| Days | จำนวนวันที่ได้รับต่อปี |

## 5. Assets (ทรัพย์สิน IT)

### ตำแหน่ง
**หน้า Assets** (SP List: `IT_Assets`)

### ประเภท Asset และฟิลด์พิเศษ

| Category | Portal URL | SSL Check |
|---|---|---|
| VM | ✅ | - |
| Software | ✅ | - |
| Certificate | ✅ | ✅ คำนวณจาก ExpiryDate อัตโนมัติ |
| Hardware | - | - |
| Network | - | - |

### AssetCode
- สร้างอัตโนมัติในรูปแบบ `IT-YYYY-NNN` เมื่อคลิก **+ เพิ่ม Asset**
- ตัวเลข NNN นับต่อจากเลขสูงสุดในปีนั้น

### SSL Days (Certificate)
- คำนวณจาก **ExpiryDate** แบบ real-time ทุกครั้งที่เปิดหน้า
- สี: 🟢 > 30 วัน / 🟡 8-30 วัน / 🔴 ≤ 7 วัน

## 6. Projects (โครงการ)

### ตำแหน่ง
**หน้า Projects** (SP List: `PM_Projects`)

### Filter โครงการในหน้าแจ้งงาน
- Dropdown เลือกโครงการมี Checkbox **"เฉพาะ Active"**
- ค่า Default = เปิด (แสดงเฉพาะโครงการ Active)
- ปิดเพื่อดูทุกโครงการ (รวม Completed / On Hold)
- โครงการที่ไม่ Active จะแสดง `[Status]` ต่อท้ายชื่อ

## 7. สถานะ Ticket

| สถานะ | ความหมาย |
|---|---|
| **Open** | Ticket ถูกสร้างแล้ว รอ Agent รับงาน |
| **In Progress** | Agent รับงานแล้วและกำลังดำเนินการแก้ไข |
| **Pending** | รอปัจจัยภายนอก เช่น รอลูกค้าตอบ / รอ Vendor / รอ Approve |
| **Resolved** | แก้ไขแล้ว รอลูกค้ายืนยันก่อนปิด |
| **Closed** | ปิดงานสมบูรณ์ ลูกค้ายืนยันหรือผ่านเวลากำหนดแล้ว |

## 8. SLA Compliance (ใน Reports)

คำนวณจาก % ของ Ticket ที่ **Resolved/Closed ทันกำหนด (DueDate)**

```
SLA% = (Ticket ที่ ResolvedDate ≤ DueDate) / (Ticket ทั้งหมดที่ปิดแล้ว) × 100
```

- Ticket ที่ยังไม่ปิดและเลยกำหนด = **Overdue** (นับแยก)
- Ticket ที่ไม่มี DueDate = ไม่นำมาคิด SLA
